// 01 — Fonctionnalités

Ce que tu peux tracker

Chaque interaction génère un événement structuré avec timestamp, coordonnées et contexte. Les données sont stockées localement dans ton navigateur.

🖱️

Clics

Position X/Y, élément ciblé, timestamp précis, profondeur dans le DOM.

📜

Scroll

Profondeur maximale atteinte, vitesse de défilement, direction.

👁️

Pages vues

URL, referrer, heure d'arrivée, résolution d'écran, langue du navigateur.

📐

Sections vues

Détection via IntersectionObserver de chaque section visible à l'écran.

// 02 — À propos du tracking

Comment ça fonctionne ?

Le web tracking consiste à intercepter les événements natifs du navigateur — click, scroll, visibilitychange — pour construire une représentation du comportement utilisateur.

Dans une implémentation réelle, ces données sont envoyées vers un serveur via des requêtes fetch ou beacon. Ici, elles sont stockées dans le localStorage pour rester 100% local et sans serveur.

Le dashboard te permet de rejouer ta session, visualiser les zones de clic, et analyser la profondeur de scroll — exactement comme le font des outils comme Hotjar, Mixpanel ou Plausible.
